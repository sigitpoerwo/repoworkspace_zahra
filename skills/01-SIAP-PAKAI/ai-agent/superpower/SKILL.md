# 🚀 Superpower - Complete Development Toolkit

All essential development workflow skills combined into one powerful toolkit.

## Included Capabilities

### 🧠 Brainstorming & Design
Use BEFORE any creative work. Explores requirements, proposes approaches, and presents designs for approval.

**When to use:**
- Starting a new feature
- Designing architecture
- Planning a refactor

### 🔍 Systematic Debugging
4-phase root cause analysis: Reproduce → Isolate → Identify → Verify

**When to use:**
- Something is broken
- Tests are failing
- Unexpected behavior

### 🧪 Test-Driven Development
Strict RED-GREEN-REFACTOR cycle for disciplined coding.

**When to use:**
- Writing new code
- Fixing bugs (write failing test first)
- Refactoring

### 👁️ Code Review
Systematic review with severity-based issue reporting.

**When to use:**
- Before merging
- After implementation
- Quality assurance

### 📋 Implementation Planning
Detailed plans with bite-sized 2-5 minute tasks.

**When to use:**
- After design approval
- Breaking down complex work
- Coordinating with team

### ✅ Verification
Pre-completion checklist to ensure quality.

**When to use:**
- Before marking done
- Before merge
- Before deployment

### 🏁 Branch Management
Complete development branch lifecycle.

**When to use:**
- Feature complete
- Ready to merge
- Cleanup needed

## Workflow Integration

### Starting New Work
1. **Brainstorm** → Get design approved
2. **Plan** → Create implementation tasks
3. **TDD** → Implement with tests
4. **Verify** → Ensure quality
5. **Review** → Get code reviewed
6. **Finish** → Merge or create PR

### Fixing Bugs
1. **Debug** → Find root cause
2. **TDD** → Write failing test, then fix
3. **Verify** → Ensure fix works
4. **Finish** → Merge fix

## Key Principles

- **Design first, code second**
- **Test first, implement second**
- **Verify before done**
- **Review before merge**
- **One thing at a time**

## 🤔 When to Archive

Move skills here when:

- ❌ **Not Useful**: Doesn't solve any real problem
- ❌ **Duplicate**: Better alternative exists
- ❌ **Outdated**: Technology or approach is obsolete
- ❌ **Too Complex**: Setup effort > benefit
- ❌ **Never Used**: Installed but never needed

---

## 📋 Archive Process

1. **Document Reason**
   - Why is it being archived?
   - What was tried?
   - Any alternatives?

2. **Move to Archive**
   ```bash
   mv path/to/skill 04-TIDAK-RELEVAN/
   ```

3. **Update INVENTARIS.md**
   - Remove from active count
   - Add to archived list

4. **Create Archive Note**
   - Document in ARCHIVE_LOG.md
   - Include date and reason

---

## 📝 Archive Log Template

```markdown
## [Skill Name]
**Archived**: [date]
**Reason**: [why archived]
**Tried**: [what was tested]
**Alternative**: [better option if any]
```

## 🔄 Restoration

If a skill becomes relevant again:

```bash
# Move back to appropriate category
mv 04-TIDAK-RELEVAN/skill-name \
   01-SIAP-PAKAI/category/
```

Update INVENTARIS.md accordingly.

---

## 💡 Tips

- Don't delete skills immediately
- Archive first, delete later if still unused
- Keep archive log for reference
- Review archived skills quarterly
